不完全定義多出力関数における FPRM の最小化

積項の共有についても考慮しながら最小化を行う。

=== 実行 ===
main.jl で、積項を共有しない場合(non_shared)と共有する場合(shared)の両方の最小化結果を出力する。
## 各ファイルについて

### README.md
このファイル。srcディレクトリ内のファイルの説明。

### makefile
main.jl を端末上で make run により、実行する用。

### main.jl
完全定義多出力関数の真理値ベクトルの入力もしくは、pla ファイルの読み込みから、指定した個数分出力をランダムに don't care に変更(不完全定義多出力関数を作成)し、
積項を共有しない場合(non_shared)と共有する場合(shared)の両方の最小化結果を出力する。

### Alg_shared.jl
多出力関数の真理値ベクトルの入力から、最小化のための重みベクトルを作成するファイル。

### exp1.jl
don't care の個数により、どのように最小 FPRM の積項数が推移していくのかを確認するための実験ファイル。

### exp2.jl
don't care の個数により、どのように最小 FPRM の積項数と、それぞれの方法による最小 FPRM 探索時間が推移していくのかを確認するための実験ファイル。
#### 探索方法
・積項を共有しない場合 (non_shared)
・積項を共有し、出力全体で極性を固定しない場合 (shared1)
・積項を共有し、出力全体で極性を固定する場合　 (shared2)

### for_println.jl
結果出力用ファイル。

### ---.pla
ベンチマーク関数のデータが入っている pla ファイル。

### read_benchmark.jl
plaファイルから真理値ベクトルを作成するためのファイル

# 使い方

ターミナル上で、make run を実行

デフォルトでは、完全定義多出力関数として、"Cmp.pla"ファイルを読み込み、出力のうち15個をランダムに don't care に変更し、不完全定義多出力関数を作成している。
読み込む pla ファイルや、直接入力する真理値ベクトル、指定する don't care の個数はファイル内を直接変更することで適宜調節できる。

## 入力後の動作(main.jl)

入力後、"Alg_share.jl"を実行し、最小化のための重みベクトルを作成。(ほとんどの実行時間はここ)

重みベクトルを作成した後は、以下の２通りの方法で最小 FPRM の探索を行う。積項を共有する場合については、出力全体で極性を固定する場合、固定しない場合の２通りから選んで探索
・積項を共有しない (find_min_ns)
・積項を共有する -出力全体で極性を固定しない (find_min_s1)
          　　 -出力全体で極性を固定 (find_min_s2)

"for_println.jl"を読み込み、以下の最小化結果を出力
・最小重み値 (最小積項数)
・極性ベクトル
・割当てベクトル
・FPRM 論理式
